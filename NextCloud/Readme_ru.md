# Nextcloud Upgrade Ansible Role

Ansible-роль для обновления существующей установки **Nextcloud** с минимальным временем простоя.

Роль автоматически загружает указанную версию Nextcloud, подготавливает новую директорию установки, переносит конфигурацию, переключает активную версию с помощью символической ссылки и выполняет обновление базы данных через `occ upgrade`.

## Выполняемые действия

* 📥 Загрузка указанной версии Nextcloud с официального сайта.
* 📦 Распаковка архива на удалённом сервере.
* ⚙️ Копирование существующей конфигурации.
* 🛠 Включение режима технического обслуживания (Maintenance Mode).
* 🌐 Остановка Apache на время обновления.
* 🔗 Подключение внешнего каталога приложений (`apps`) через символическую ссылку.
* 🔄 Переключение активной версии Nextcloud на новую.
* 🚀 Перезапуск PHP-FPM.
* 🗄 Выполнение обновления базы данных (`occ upgrade`).
* ✅ Отключение режима обслуживания.
* 🧹 Очистка временных файлов.

## Требования

Перед использованием роли необходимо:

* Ansible **2.12** или выше;
* установленный **Apache**;
* установленный **PHP-FPM**;
* установленный **PHP CLI**;
* существующая установка Nextcloud;
* пользователь веб-сервера (`www-data`);
* отдельный каталог приложений (`apps`);
* права на выполнение команд с `become`.

## Используемые переменные

| Переменная | Значение по умолчанию | Описание |
| --- | --- | --- |
| `nextcloud_version` | `34.0.0` | Версия Nextcloud для установки |
| `nextcloud_user` | `www-data` | Пользователь, от имени которого выполняются команды `occ` |
| `nextcloud_base_path` | `/var/www` | Базовый каталог установки |
| `nextcloud_current_path` | `/var/www/nextcloud` | Символическая ссылка на текущую версию |
| `nextcloud_new_path` | `/var/www/{{ nextcloud_version }}` | Каталог новой версии |
| `apps_path` | `/var/www/nextcloud_apps` | Каталог внешних приложений |
| `download_url` | Формируется автоматически | URL для загрузки архива Nextcloud |
| `php_service` | `php8.3-fpm.service` | Имя службы PHP-FPM |

## Пример playbook

```yaml
- name: Обновление Nextcloud
  hosts: host_group
  become: true

  roles:
    - nextcloud
```

## Запуск

Выполнить роль можно командой:

```bash
ansible-playbook nextcloud.yml --tags update_service
```

## Порядок выполнения обновления

Во время работы роль выполняет следующие действия:

1. Загружает архив с указанной версией Nextcloud.
2. Распаковывает архив.
3. Перемещает новую версию в каталог установки.
4. Копирует каталог `config`.
5. Включает режим технического обслуживания (`maintenance mode`).
6. Останавливает Apache.
7. Назначает владельца и необходимые права доступа.
8. Заменяет встроенный каталог `apps` символической ссылкой.
9. Переключает символическую ссылку `/var/www/nextcloud` на новую версию.
10. Запускает Apache.
11. Перезапускает PHP-FPM.
12. Выполняет команду `occ upgrade`.
13. Отключает режим технического обслуживания.
14. Удаляет временный архив.

## Предполагаемая структура установки

Роль рассчитана на следующую схему размещения файлов:

```text
/var/www/
├── nextcloud -> /var/www/34.0.0
├── 34.0.0/
├── nextcloud_apps/
└── nextcloud_data/
```

Каталог `nextcloud` является символической ссылкой на активную версию приложения.

## Важно

Перед запуском обновления рекомендуется:

* создать резервную копию базы данных;
* сохранить каталог `data`;
* убедиться, что на сервере достаточно свободного места;
* протестировать обновление на тестовом сервере.

> **Внимание:** роль предназначена **только для обновления существующей установки Nextcloud** и не выполняет первоначальную установку приложения.
